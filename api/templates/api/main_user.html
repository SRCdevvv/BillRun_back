/api/sms_confirm

유저가 존재하지 않을 경우 -> 회원가입. 이메일 인증으로 이동해주세요
유저가 존재할 경우 -> 로그인이 됩니다.

/api/email

phone, community, email 작성해서 POST 하면 회원이 생성됩니다.

/api/terms

유저의 약관동의 일자를 받습니다. 유저 생성과 동시에 데이터 저장하면 될 것 같습니다!
유저 id를 뒤에 붙이면 해당 유저의 동의 일시가 나옵니다.

특정 유저에 대한 리뷰 작성 페이지
